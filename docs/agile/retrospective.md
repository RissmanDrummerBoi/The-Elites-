### What Could Be Improved

| Category | Feedback | Action Item |

3. **Git branching:** Establish clear branching strategy (feature branches from main)

### What Could Be Improved

| Category | Feedback | Action Item |
|----------|----------|-------------|
| **Error Handling** | Registration error messages were unclear to users. | Improve error message clarity in Sprint 2. |
| **Mobile Responsiveness** | Dashboard not fully responsive on mobile devices. | Kelly to add responsive design improvements in Sprint 2. |

### Lessons Learned

1. **Error messages:** User-friendly error messages are critical for good UX
2. **Responsive design:** Test on mobile devices early, not at the end
3. **API documentation:** Update documentation as part of Definition of Done
4. **Research feedback:** Involve Jabu earlier in the sprint for faster validation
