## Project Management Approach

The project will follow an **Agile (Scrum) approach**. Development will be carried out in short iterations (sprints), allowing continuous feedback, testing, and improvement.

Regular team meetings (Scrums) will be held to:
- Track progress  
- Identify challenges  
- Adjust tasks when necessary  

This approach is suitable because system requirements may evolve based on user feedback, and Agile allows flexibility and continuous improvement.

## Risk Management Plan

- **Risk:** Late submission or missed deadlines  
  → **Mitigation:** Weekly meetings, task tracking, and clear deadlines for each member  

- **Risk:** Poor communication within the team  
  → **Mitigation:** Use WhatsApp group and regular Scrum meetings to ensure continuous communication  

- **Risk:** Integration issues between frontend and backend  
  → **Mitigation:** Continuous testing and early integration of system components  

- **Risk:** Unclear or changing requirements  
  → **Mitigation:** Conduct proper research and validate requirements with users early  

- **Risk:** Incomplete or incorrect user input (e.g., missing details in complaints)  
  → **Mitigation:** Implement input validation and structured forms in the system

## Technology Stack

The selected technology stack is based on simplicity, efficiency, and scalability for the project.

- **Frontend:** React.js  
  → Used to build a responsive and interactive user interface. React supports component-based development, making the UI easier to manage and scale.

- **Backend:** Python (Flask)  
  → Flask is lightweight and suitable for small to medium-sized applications. It allows quick development and easy integration with the frontend.

- **Database:** MySQL  
  → Used to store structured data such as user information, complaints, and system records. It is reliable and easy to manage.

- **Version Control:** GitHub  
  → Used for collaboration, code management, and tracking changes. It helps the team work together efficiently and maintain version history.
